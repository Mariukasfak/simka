## 2024-05-23 - [Prevented Parent Re-renders on Drag]
**Learning:** React components re-rendering on every mouse move (even if throttled) can be a significant performance bottleneck, especially if the parent component is complex.
**Action:** Use local state for high-frequency updates (like dragging) and only sync with the parent state on completion (drag end). This isolates the re-renders to the leaf component.
## 2026-05-08 - O(N^2) Array.find to O(N) Map refactor in Analytics API\n**Learning:** Found an (N^2)$ aggregation bottleneck in `app/api/admin/analytics/route.ts` where `Array.find` was used inside a `reduce` loop to group data by date. JavaScript `Map` structures naturally preserve insertion order, making them perfect for this exact scenario—allowing an (N)$ refactor without needing an additional sorting step at the end.\n**Action:** When aggregating or grouping datasets by keys (e.g., grouping by date or ID), default to using a `Map` or an object-based lookup instead of nested `find` or `filter` methods within a loop. This pattern avoids severe performance degradation on large datasets.
