## 2024-05-23 - [Prevented Parent Re-renders on Drag]
**Learning:** React components re-rendering on every mouse move (even if throttled) can be a significant performance bottleneck, especially if the parent component is complex.
**Action:** Use local state for high-frequency updates (like dragging) and only sync with the parent state on completion (drag end). This isolates the re-renders to the leaf component.
## 2026-04-18 - [Parallelize Independent Supabase Queries]\n**Learning:** Sequential Supabase data fetching (waterfall queries) in API routes is a common backend performance bottleneck. Because Supabase uses REST underneath, wrapping independent query promises in `Promise.all()` significantly reduces latency from O(T1 + T2 + T3) to O(max(T1, T2, T3)). Furthermore, Supabase fetch methods resolve gracefully with `{ data, error }` rather than throwing exceptions, meaning they perfectly slot into a single `Promise.all` array where errors can simply be evaluated synchronously afterward.\n**Action:** When an API route fetches multiple separate datasets (e.g., aggregate orders, top products, and time-series metrics), destruct them from a `Promise.all()` block concurrently rather than awaiting them one-by-one.
