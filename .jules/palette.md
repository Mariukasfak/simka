## 2024-03-25 - Standardize Modal/Tooltip Close Button Accessibility
**Learning:** In the `EnhancedDesignCanvas` component (and likely other custom modals/tooltips), icon-only close buttons (like Lucide `<X>`) lacked consistent accessibility properties and focus states. Screen readers either misread them or ignored them, and keyboard users lacked clear visual feedback. Since the app is in Lithuanian, aria-labels must also be localized.
**Action:** When adding close buttons to custom absolute/fixed positioned overlays, always apply `aria-label="Uždaryti"` (or more specifically, e.g., "Uždaryti patarimą"), add `aria-hidden="true"` to the inner SVG icon, and explicitly include `focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-accent-500 rounded` classes for keyboard navigation.
