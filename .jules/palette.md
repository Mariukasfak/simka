## 2024-03-24 - Making Icon-only Buttons Accessible
**Learning:** Custom icon-only buttons (like the `X` buttons in `EnhancedDesignCanvas.tsx` for tooltips/modals) often lack `aria-label`s, have purely decorative icons that screen readers announce awkwardly if not hidden with `aria-hidden="true"`, and completely lack keyboard focus styles since they don't use standard `<Button>` components.
**Action:** When implementing standalone/custom `<button>` elements that only contain an icon, always add an explicit `aria-label` describing the action, add `aria-hidden="true"` to the inner SVG/icon component, and add explicit Tailwind focus ring classes (`focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-accent-600 rounded`) to guarantee keyboard navigation visibility.
