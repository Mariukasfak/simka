## 2024-04-10 - Accessibility for Icon-only Buttons
**Learning:** Icon-only buttons using Next.js/Lucide component sets often miss critical accessibility attributes, such as `aria-label` for screen readers and `focus-visible` styles for keyboard navigation. Specifically, raw `<button>` elements in this project (e.g., in `EnhancedDesignCanvas.tsx`) don't have default focus styles and need manual application of `focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-accent-600 rounded`, alongside an `aria-hidden="true"` attribute on the internal SVG to prevent redundant screen reader announcements.
**Action:** When adding or updating icon-only raw `<button>` elements in this project, explicitly add `aria-label`, ensure proper `focus-visible` styles matching the context, and pass `aria-hidden="true"` to the internal `<Icon>` element.
